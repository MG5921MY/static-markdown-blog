## 项目主题整体评价
项目已有 4 个内置主题（graphite / aurora / paper / mono）+ 1 套 base.css，整体水平在「能看、功能完整、token 体系规整」这一档，但**距离"精美/有设计感"仍有明显距离**。问题不是大方向错，而是**细节颗粒度不够 + 风格符号太通用**。

## 当前的 AI 感/通用感来自哪里
1. **首屏 Hero 视觉过于"通用"**
   - 圆形高斯模糊光斑 + 单行 kicker + 大字标题，几乎是 2023 后所有 AI 站和 SaaS 落地页的默认配方
   - 标题里 3 行"静态 / Markdown / 博客"硬换行是典型 AI 文案
   - aurora 主题是浅紫渐变 + 紫蓝高光，**这正是 SaaS Landing Page 模板里最泛滥的色板**
2. **圆角语言单一**
   - 全站 999px 药丸（filter、按钮、tag、nav-link、toc 收起、kicker）一统到底
   - 加上 `--radius-lg: 22px`、`--radius-xl: 30px` 这种偏圆润的 token，让所有主题都偏"幼态"
   - mono 主题写"开发者黑白极简"却用 `border-radius: 4/8/12/16`，没有真正的方正感
3. **配色 / 字体缺少锚点**
   - 4 套主题里只有"紫蓝"是真正有性格的（aurora 默认色），其他 3 套都是中性灰
   - 字体退化到 PingFang / Microsoft YaHei，**没有任何 display font 拉差异**
   - mono 主题号称开发者极简，但字体却是 Segoe UI，没有 mono 字符风格的标题
4. **节奏与细节**
   - 卡片间距、阴影、border 全是 base.css 一套参数，4 个主题的"差异"基本只剩颜色
   - 阅读进度条 / 目录 / 返回顶部这些交互堆得偏多，但视觉上没有"克制的优雅"
   - 链接列表（links）头像用了大写字母 + 药丸 + 描述，但排版上还是 "Material 卡片" 那一套

## 改进方向（按性价比排序）

### A. 把"通用 SaaS 模板感"换掉
- **Hero 不要再做右上角光晕 + 大字**。改成"杂志/刊物式"或"档案柜式"或"手记式"三种不同 hero 范式
- **首屏标题去掉硬换行**，让 2~3 句长短不一的小字组成
- **kicker 不要做"药丸徽章"**，改成序号 / 章节号 / 日期型标签
- **每个主题给一个"signature element"**（签名元素），而不是只换颜色：
  - graphite：一根贯穿的横线 / 表格分隔线 / 等宽编号
  - aurora：渐变但只用在 1~2 处 + 大量留白
  - paper：纸边阴影 + 落款式署名 + 行高更宽
  - mono：方角、密集 grid、代码块当装饰

### B. 字体与字重
- 接入真实可用的 display 字体（不能用联网字体就靠系统字体组合）
- 例如 graphite 用 "Cascadia Code / Consolas" 做 display
- paper 用衬线（"Songti SC / Source Han Serif"）做正文
- mono 强制所有标题用 monospace

### C. 圆角 / 间距 token
- 主题和主题之间至少在 radius / space / shadow 上有显著差异
- mono 强制 `border-radius: 0`
- paper 偏小圆角 + 暖阴影
- aurora 偏大圆角 + 软光晕
- graphite 默认但更克制（12~16px 而非 22~30px）

### D. 单一主题的深度
- 不要平均用力打磨 4 套，**默认主题做精**，其他主题只给"明显不同"够用
- 主页推荐主推一个，配置里改默认

## 立即能改的小项（不动结构）
- Hero 主标题断行换成长短句
- kicker 徽章改序号格式（如 "01 / Notes"）
- 列表卡片加 hover 时的"细节副线"（左 border、underline、序号）
- 链接页加 favicon 占位（用首字母生成 SVG）
- footer 加一行站点副标题（中文/英文）
- 滚动条、selection 颜色、focus ring 改用主题主色
